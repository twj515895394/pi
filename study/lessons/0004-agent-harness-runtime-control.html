0004：AgentHarness 是 Runtime 控制层

状态：已完成。对应参考资料：agent-harness-runtime-control.md

本课目标

理解 AgentHarness 为什么不是简单封装 Agent，而是把模型、工具、消息、权限、事件、上下文、Session、Compaction、Branch Summary 和 UI/CLI/SDK 连接起来的 Runtime 控制层。

核心分层

层级类比职责
AgentLoop核心工作流函数当前 run 内模型-工具闭环
Agent有状态 Service保存 messages / tools / streaming 状态
AgentHarness应用运行容器Session、模型、工具、资源、Hook、队列、Provider、长期上下文和事件治理

createTurnState 与 createContext

createTurnState() 组装富状态运行快照；createContext() 把它压缩成 AgentLoop 所需的最小上下文。

TurnState:
  session context
  resources
  session metadata
  tools / activeTools
  systemPrompt
  model
  thinkingLevel
  streamOptions

AgentContext:
  systemPrompt
  messages
  tools

createLoopConfig 的作用

createLoopConfig() 是 Harness 到 AgentLoop 的适配器 / 注入器。

注入点Harness 能力
transformContextcontext hook：上下文治理
beforeToolCalltool_call hook：权限、审批、风控
afterToolCalltool_result hook：脱敏、压缩、错误标准化
prepareNextTurnflush session writes，重建 TurnState
getSteeringMessages运行中插话队列
getFollowUpMessages后续消息队列

企业清关 Agent Harness 示例

入口控制：用户请求合法性、租户、用户身份
上下文控制：历史消息、当前清关单、业务上下文
模型控制：模型选择、thinkingLevel、provider 参数
工具控制：当前角色可用哪些清关工具
权限控制：tool_call hook 执行前审批、权限、风控
结果控制：tool_result hook 执行后脱敏、压缩、错误标准化
会话控制：消息落库、save point、恢复上下文
审计控制：请求日志、工具调用日志、provider 调用记录
UI 控制：message/tool/queue/save_point/settled 事件推送
长期上下文控制：compaction 与 branch summary
