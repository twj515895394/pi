0005：Session Tree 与 Compaction

状态：已讲解，待理解检查。对应参考资料：session-tree-and-compaction.md

本课目标

理解 Pi 的 Session 为什么不是简单聊天记录，而是包含 message、model_change、active_tools_change、compaction、branch_summary 等节点的运行历史树；理解 Compaction 和 Branch Summary 如何服务长上下文与分支会话。

一句话结论

Session Tree 保存 Agent 的可导航运行历史；Compaction 把旧历史折叠成摘要检查点；Branch Summary 把离开的分支压成移交说明。

Session Tree 数据模型

Entry是否进入 LLM Context职责
message是用户、助手、工具结果、bash 等消息
custom_message是Extension 注入给模型看的上下文
branch_summary是切换分支时保留被离开分支的摘要
compaction间接是生成 compactionSummary，替代旧历史进入上下文
model_change否恢复当前 branch 的模型选择
thinking_level_change否恢复当前 branch 的推理强度
active_tools_change否恢复当前 branch 的可用工具集合
custom否Extension 私有状态
label / session_info / leaf否书签、展示名、当前指针

buildSessionContext 流程

pathEntries = session.getBranch(currentLeaf)

for entry in pathEntries:
  update thinkingLevel / model / activeTools
  remember latest compaction

if compaction exists:
  messages = [compactionSummary]
           + messages from firstKeptEntryId to compaction
           + messages after compaction
else:
  messages = all message/custom_message/branch_summary on path

重点：Compaction 不删除旧 JSONL entry，只改变“下次给 LLM 的上下文视图”。

Compaction 流程

AgentHarness.compact()
  -> session.getBranch()
  -> prepareCompaction()
       -> estimateContextTokens()
       -> findCutPoint(keepRecentTokens)
       -> collect messagesToSummarize
       -> collect turnPrefixMessages if split turn
       -> extract fileOps
  -> session_before_compact hook
  -> compact()
       -> generateSummary()
       -> optional generateTurnPrefixSummary()
  -> session.appendCompaction()
  -> emit session_compact

切点规则

规则原因
保留最近 keepRecentTokens最近工作最好原样保留
预留 reserveTokens给摘要 prompt 和模型输出留空间
不切 toolResulttool result 必须和 tool call 保持语义配对
允许 split turn单轮过大时摘要 turn prefix，保留后缀继续执行
重复压缩携带 previousSummary防止多次压缩丢掉更早历史

Branch Summary 流程

AgentHarness.navigateTree(targetId, summarize=true)
  -> oldLeafId = session.getLeafId()
  -> collectEntriesForBranchSummary(oldLeafId, targetId)
       -> find common ancestor
       -> collect old leaf back to ancestor
  -> session_before_tree hook
  -> optional generateBranchSummary()
  -> session.moveTo(newLeafId, summary?)
  -> emit session_tree

信贷审批 Agent

Session Tree = 审批过程审计树
Compaction = 长审批链路检查点摘要
Branch Summary = 方案切换时的分支移交说明
Leaf = 当前正在推进的审批路径
